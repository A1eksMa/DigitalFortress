## 1. Выбор хостинг-провайдера: ABLENET

### 1.1. Обоснование выбора

Провайдер ABLENET был выбран по следующим причинам:

*   **Доступность:** Официальный сайт ABLENET был свободно доступен с территории РФ, не попадая под текущие ограничения.
*   **Язык интерфейса:** Интерфейс имел поддержку английского языка, что упрощало навигацию и управление (хотя и не все страницы были переведены).
*   **Стоимость и характеристики:** Относительно небольшая стоимость сервера (приблизительно $8-10/месяц) при достаточно хороших технических характеристиках для минимального сервера, соответствующих требованиям, изложенным в `entry-node-hosting-requirements.md`.
*   **Пробный период:** Предоставление бесплатных двух недель пробного периода для тестирования сервиса.
*   **Репутация:** Положительные отзывы и долгий срок работы компании на рынке (25+ лет).
*   **Соответствие требованиям:** Это был единственный из чисто японских хостеров, который полностью соответствовал всем заявленным требованиям.

### 1.2. Резервный провайдер

В качестве резервного варианта рассматривался [ConoHa (GMO Internet Group)](https://www.conoha.jp/en/). Однако на одном из этапов регистрации возникли проблемы с доступом к сайту. Несмотря на это, ConoHa предлагает почасовую оплату, что является преимуществом, и может быть рассмотрен в будущем.

## 2. Характеристики инстанса Entry-ноды

### 2.1. Основные параметры

| Параметр            | Значение                                 |
| :------------------ | :--------------------------------------- |
| Service Plan        | V1 SSD plan                              |
| Memory              | 1.5 GB                                   |
| vCPU(s)             | 2 Core(s)                                |

### 2.2. Сетевые параметры

| Параметр            | Значение            |
| :------------------ | :------------------ |
| IPv4 Adress         | 202.223.48.9        |

### 2.3. Особенности настройки

*   **Установка с нуля:** Провайдер ABLENET предоставляет возможность настройки сервера с полного нуля, включая выбор операционной системы.
*   **Операционная система:** На сервере установлена Ubuntu 22.
*   **Минимальные настройки безопасности:**
    *   Настроен доступ по SSH с использованием ключей.
    *   Установлен и настроен файрвол UFW.
    *   Создан пользователь с правами `sudo`.
    *   Открыты порты 22 (SSH) и 443 (HTTPS).
